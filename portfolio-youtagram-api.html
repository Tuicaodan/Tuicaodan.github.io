GraphQL API Endpoint for the Youtagram web app

Project:

The project is to create a GraphQL API endpoint for the Youtagram web app. The Youtagram web app will send requests to the endpoint. The server will fetch data depends on the requests from a MongoDB database deployed on MongoDB Altas cloud.

The is an excellent practical project to learn how to implement Node.js, GraphQL, DatatLoader, MongoDB and JWT together.

Project Difficulties:

The most difficult part was learning how to write the GraphQL API endpoint. Unlike the usual REST API, GraphQL allows users to dictate the exact data they need from the server. It avoids let users send multiple API requests or have to receive a complete set of data that contains unnecessary information. So how to structure the GraphQL data type, resolver, queries, and mutations is the most changeable part.

Another difficulty was solving GraphQL's N+1 issue. The cost of sending requests to a database to fetch data is expensive. Because of the nested structure of GraphQL requests, there will be lots of duplicate database requests. Need to find the optimal way to solve this issue.

The solution:

Leaning a new skill set is always hard. But this is the fun part. Online tutorials and official docs are the best resources. In a short time, I was able to understand and write GraphQL types, resolvers, queries and mutations.

I also found the DataLoader was the answer to the N+1 issue. DataLoader will collect the requests and send the request in batch to the database. I was able to implement DataLoader in the project as well.
